# Архив плана: Исправление смещения и отсекания при экспорте карты (map-export)

*Этот файл содержит архивные данные плана, который был успешно выполнен 2026-07-20 перед началом новой задачи исследования.*

## Описание проблемы
В компоненте `map-export` при экспорте фрагмента карты результирующее изображение оказывалось сдвинутым относительно рамки видоискателя на экране, из-за чего часть изображения отсекалась.
Это было вызвано тем, что расчет экранных координат видоискателя относительно центра карты рассчитывался как `centerX = contW / 2` и `centerY = contH / 2`. Это предполагало, что рамка видоискателя находится ровно по центру контейнера карты. Однако:
1. Контейнер карты смещен по вертикали из-за верхнего тулбара (`app-map-toolbar`) и статус-бара (`app-status-bar`).
2. При открытых сайдбарах (левый `app-map-sidebar`, правый `app-map-layers-panel`) ширина контейнера карты уменьшается, сдвигая его центр по горизонтали.
3. Рамка видоискателя центрирована относительно всего экрана (`fixed`, `100vw` х `100vh`).

## Реализованное решение
1. **Устранение Race Condition:** Считывание реактивных размеров `printSizeMm()`, `resultPixels()` и DOM-параметров рамки перенесено в самое начало метода `generateExport()`, строго до изменения реактивного флага `isGenerating`.
2. **Точный расчет экранных координат рамки:** Использован метод `getBoundingClientRect()` для `.viewfinder-box` и контейнера карты (`mainMap.getContainer()`).
3. **Географический центр без искажений:** Центр `exportCenter` рассчитывается путем проецирования точной центральной точки экрана рамки `unproject([(x1 + x2)/2, (y1 + y2)/2])` вместо среднего арифметического угловых координат.
4. **Инвариантный расчет расстояний (Хаверсинус по 3 точкам):** Внедрен приватный метод `getGeodistance()` (формула Хаверсинуса для сферы Земли). Ширина `widthMeters` рассчитывается как геодезическое расстояние между разпроецированными левым верхним `p_tl` и правым верхним `p_tr` углами рамки.
5. **Сохранение пропорций кадрирования (aspect ratio):** Высота печатного листа `heightMm` вычисляется строго пропорционально `aspect` рамки видоискателя на экране (`widthMmResult / aspect`). Это делает результирующий Canvas геометрически подобным видоискателю, исключая обрезание по горизонтали.
6. **Компенсационный сдвиг охвата (+10% по краям):**
   * В режиме заданного масштаба: `widthMeters` умножается на коэффициент `1.10`.
   * В режиме "Авто": уровень `exportZoom` корректируется на `-0.1375` уровней.
7. **Синхронизация разрешения (WebGL-ограничение в UI):** Расчет WebGL-лимита (`maxWebGLSize` на основе `gl.MAX_TEXTURE_SIZE`) и безопасного сжатия `safeScale` перенесен непосредственно в реактивный computed-сигнал `resultPixels()`.
8. **Компенсация безопасного масштаба (safeScale):** Если `safeScale < 1.0`, зум скрытой карты уменьшается на `Math.log2(safeScale)`, сохраняя исходный географический охват кадрирования.
9. **Вызов map.resize() и принудительный layout (reflow):** После монтирования `hiddenDiv` к DOM-дереву синхронно считывается `hiddenDiv.offsetHeight` и делается микропауза 50мс.
10. **Обновление расчета exportZoom в режиме "Авто":** Используется реальная ширина рамки `vfRect.width` для расчета масштабного коэффициента.
