## 🚀 1. Рефакторинг логики переключения карт на основе Signals

### Контекст и проблема
Ранее переключение карт производилось прямыми вызовами `renderMap` из HTML-шаблона `map-canvas.component.html` по нажатию на две сырые нестилизованные кнопки. Это нарушало разделение ответственности и мешало отображению активной карты в интерфейсе.

### Реализованное решение
1. **Состояние в ViewModel:** В [map.viewmodel.ts](file:///F:/Vanya/topos/src/app/features/map-view/viewmodels/map.viewmodel.ts) добавлен сигнал `activeMapId` со значениями `'map1' | 'map2'`, отслеживающий выбранную подложку:
   ```typescript
   readonly activeMapId = signal<'map1' | 'map2'>('map1');
   ```
2. **Реактивный рендеринг в Canvas:** В [map-canvas.component.ts](file:///F:/Vanya/topos/src/app/features/map-view/components/map-canvas/map-canvas.component.ts) добавлен реактивный `effect`, отслеживающий изменения `activeMapId` и разрешение `mapContainer`:
   ```typescript
   constructor() {
     effect(() => {
       const activeId = this.vm.activeMapId();
       const container = this.mapContainer();
       if (container) {
         const targetMap = this.mapsUrls[activeId];
         this.renderMap(targetMap.url, targetMap.type);
       }
     });
   }
   ```
   Удален прямой вызов из `ngAfterViewInit`, который теперь выполняется автоматически при инициализации контейнера.

## 📐 5. Интерактивное трансформирование иконок (Bounding Box + Rotate/Resize)

Реализована интуитивно понятная система прямого трансформирования тактических иконок на карте в стиле графических редакторов:

1. **Рамка трансформирования (Bounding Box):**
   - Вокруг выделенного точечного знака динамически отображается пунктирная рамка с заливкой `.transform-box` ([map-canvas.component.html](file:///F:/Vanya/topos/src/app/features/map-view/components/map-canvas/map-canvas.component.html)).
   - Позиция, размер и угол наклона рамки проецируются из геоданных в пиксели экрана с помощью метода `map.project` и CSS-свойств `transform` / `ngStyle` ([map-canvas.component.ts](file:///F:/Vanya/topos/src/app/features/map-view/components/map-canvas/map-canvas.component.ts)).
   - Рамка автоматически позиционируется, вращается и масштабируется при панорамировании, зуме, повороте и наклоне карты благодаря прослушиванию событий MapLibre GL JS.

2. **Масштабирование (Resize):**
   - На углах рамки расположены 4 круглые ручки `.transform-handle` для изменения размера (NW, NE, SW, SE).
   - При перетаскивании ручки вычисляется расстояние от курсора до центра знака. Масштаб (`size`) пересчитывается пропорционально изменению расстояния от центра и обновляется в реальном времени, с ограничением от `0.03` до `0.25`.

3. **Вращение (Rotate):**
   - Над верхней гранью рамки выведена линия-ножка с круглым рычагом вращения.
   - Зажатие и движение мыши вокруг центра знака считывает полярный угол вектора направления и транслирует его в угол поворота (`angle` от `0` до `359` градусов).
   - **Исправление смещения координат (Client Offset Fix):** Для устранения дефекта медленного вращения (когда для поворота на пару градусов требовалось делать полный круг мышью вокруг знака) формулы вычисления углов и расстояний переведены в единую систему координат Client Viewport. Пиксельные координаты центра знака, проецируемые из MapLibre (`map.project`), теперь корректируются с учетом отступов контейнера карты на странице (`getBoundingClientRect().left` и `top`), что полностью синхронизировало курсор мыши и поворот рамки 1:1.

4. **Стили и импорты:**
   - Интегрирована директива `NgStyle` в standalone-импорты компонента холста.
   - Описаны стили курсоров (`nwse-resize`, `nesw-resize`, `grab`/`grabbing`) и подсветка элементов управления Persian Blue ([map-canvas.component.css](file:///F:/Vanya/topos/src/app/features/map-view/components/map-canvas/map-canvas.component.css)).

## 🗺️ 8. Замена и перепаковка растровой основы советского Генштаба (SQLite ➔ MBTiles ➔ PMTiles)

Решена проблема низкого качества растровой подложки за счет автоматической конвертации и замены PMTiles-архива на высококачественную карту из базы данных BigPlanet/RMaps `GenshtabBelarus.sqlitedb`:

1. **Анализ исходного формата SQLite:**
   - База данных `sqlitedb` представляет собой SQLite с таблицей `tiles` (колонки `x`, `y`, `z`, `s`, `image`).
   - Выявлено, что масштабный коэффициент в БД инвертирован относительно стандартного Google Web Mercator: фактический зум рассчитывается по формуле `zoom = 17 - z`.
   - Координаты Y хранятся как стандартный XYZ, а не TMS.

2. **Математическая конвертация и перепаковка (MBTiles):**
   - Разработан Python-скрипт `convert_genshtab.py`, который считывает все 37 578 тайлов (формат JPEG) из `GenshtabBelarus.sqlitedb`.
   - При копировании в новую базу MBTiles скрипт корректирует зум (`zoom = 17 - z`) и инвертирует Y для соответствия стандарту TMS: `tile_row = (1 << zoom) - 1 - y`.
   - Сформирован корректный MBTiles-архив `GenshtabBelarus_correct.mbtiles` размером 2.1 GB.

3. **Сжатие в PMTiles и интеграция:**
   - С помощью утилиты `pmtiles.exe` база MBTiles сжата в автономный файл `GenshtabBelarus.pmtiles`.
   - Файл скопирован по пути `src-tauri/assets/belarus_topomap_200k.pmtiles`, перезаписав старый низкокачественный растр.
   - Благодаря сохранению имени файла, новая высококачественная растровая основа подгружается Tauri WebView автоматически без правок JS-кода.

4. **Исправление исчезновения и размытия растра (Zoom Limits & Blurring Fix):**
   - **Проблема 1 (Белые области):** При указании диапазона `minzoom: 0` и `maxzoom: 18` MapLibre пыталась напрямую запрашивать несуществующие тайлы крайних уровней у PMTiles-протокола. Поскольку протокол возвращал пустоту, при отдалении карта становилась белой (за исключением кэшированных тайлов).
   - **Проблема 2 (Размытие текста и линий):** По умолчанию MapLibre считает размер растровых тайлов равным `512px` и использует билинейное сглаживание (`linear`). Так как исходные тайлы Генштаба имеют размер `256px`, они искусственно растягивались в 2 раза на целочисленных зумах, а при незначительном сдвиге масштаба превращались в размытую «мыльную» сетку, делая тонкие изолинии и надписи нечитаемыми.
   - **Решение в [map-canvas.component.ts](file:///F:/Vanya/topos/src/app/features/map-view/components/map-canvas/map-canvas.component.ts):**
     - В источник `'belarus-data'` прописаны точные физические границы тайлов в архиве: `minzoom: 8`, `maxzoom: 13`. Это исключило отправку некорректных запросов к протоколу и включило правильный математический даунскейлинг (для зумов < 8) и оверскейлинг (для зумов > 13) силами самой MapLibre, устранив белые области.
     - Добавлен параметр `tileSize: 256` для вывода тайлов в оригинальном разрешении 1:1.
     - В свойства отрисовки слоя `'raster-layer'` добавлено свойство `'raster-resampling': 'nearest'` (ближайший сосед). Теперь при любых промежуточных масштабах (например, 1:200 000 / Zoom 11.2) сетка дорог, рельеф и шрифты советских топокарт остаются идеально резкими и читаемыми без интерполяционного размытия.
     - **Исправление краша векторной карты:** Так как `tileSize` и жесткие границы зума `minzoom/maxzoom` ломали парсинг и вычисление стилей для векторных тайлов (вызывая ошибку `recalculate of null` в ядре MapLibre GL JS), настройки источника `'belarus-data'` разделены: указанные параметры теперь применяются исключительно при `type === 'raster'`, а в режиме `'vector'` используются стандартные лимиты `minzoom: 0` и `maxzoom: 14` без параметра `tileSize`.


5. **Ограничение масштаба отдаления (minZoom Limit):**
   - **Задача:** Заблокировать отдаление карты мельче масштаба 1:5 000 000.
   - **Решение:** В [map-canvas.component.ts](file:///F:/Vanya/topos/src/app/features/map-view/components/map-canvas/map-canvas.component.ts) в опциях инициализации карты `new maplibregl.Map` добавлено свойство `minZoom: 6.48`. Данный уровень зума строго соответствует пределу масштаба 1:5 000 000 (в 1 см - 50 км). В [map-scale.const.ts](file:///F:/Vanya/topos/src/app/features/map-view/consts/map-scale.const.ts) добавлены новые пресеты шкал (1:1 000 000, 1:2 000 000, 1:5 000 000) для корректного отображения и выбора данных масштабов в интерфейсе.

## ⛰️ 9. Автоматическое DEM-ориентирование фортификации по изолиниям склонов (Terrain-Aligned Placement)

Реализован алгоритм автоматического ориентирования тактических инженерных сооружений (ДОТ, ДЗОТ, укрытия техники, окопы для танков/БМП) фронтом по направлению склона рельефа местности:

1. **Сервис высот `TerrainService`:**
   - Сервис [terrain.service.ts](file:///F:/Vanya/topos/src/app/features/map-view/services/terrain.service.ts) считывает векторный GeoJSON-файл `public/contours.geojson` при первом обращении.
   - Разработан парсер геометрий типа `MultiLineString` (так как в contours.geojson изолинии сгруппированы по высотным отметкам в мульти-линии).
   - Для ускорения расчетов применены AABB-фильтры (Axis-Aligned Bounding Box) для каждого элемента изолинии с буфером поиска ~2 км.

2. **Геодезический расчет направления склона:**
   - Находится ближайший сегмент изолинии к точке размещения объекта.
   - Вычисляется вектор нормали к этому сегменту.
   - Вдоль нормали в обе стороны на расстоянии 50 м замеряются высоты ближайших изолиний. Направление с наименьшей высотой принимается за вектор спуска (Downhill).
   - Вычисляется дирекционный угол (азимут) с косинусной коррекцией Mercator-проекции по широте: `dx_meters = dx * cos(lat)`.

3. **Интеграция в жизненный цикл объектов (UX):**
   - **Click placement:** При установке нового сооружения (имя которого начинается на `fort_`) оно мгновенно вычисляет экспозицию склона и разворачивается в сторону долины.
   - **Drag release:** При перетаскивании знака по карте в момент отпускания мыши (MouseUp) его азимут автоматически перевычисляется и корректируется под новый рельеф ([tactical-map.service.ts](file:///F:/Vanya/topos/src/app/features/map-view/services/tactical-map.service.ts)).
   - **Manual override:** В панель свойств добавлен ручной триггер — кнопка «⛰ Ориентировать по рельефу» ([symbol-properties.component.html](file:///F:/Vanya/topos/src/app/features/map-view/components/symbol-properties/symbol-properties.component.html)), позволяющая в 1 клик сбросить ручной угол и сориентировать элемент по горизонталям.
