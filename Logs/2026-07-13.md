# Журнал изменений - 2026-07-13

> Для будущего Gemini: лог изменений проекта за этот день.

- **Инициализация проекта и настройка 3-уровневой памяти**:
  - Создана базовая структура OKF по стандарту Antigravity CLI.
  - Инициализированы файлы `index.md`, `log.md`, `_GEMINI.md`, `MEMORY_TOPOS.md`, `.agents/user.md`, `.agents/soul.md` и `plan.md`.
- **Детализация стилей карты OpenMapTiles**:
  - В файле `map-view.ts` расширен массив слоев (layers) до 37 детальных слоев по спецификации OpenMapTiles.
  - Добавлены типы зон (лес, трава, песок, парки, промзоны, жилые, военные, больницы, школы).
  - Спроектирована иерархия дорог (motorway, trunk, primary, secondary, tertiary, minor, service, track, path) с красивой обводкой (casing) для слияния пересечений на тактическом темном фоне.
  - Добавлены 3D здания (`fill-extrusion`) на крупных масштабах (zoom >= 15) и 2D здания на меньших.
  - Реализованы локализованные текстовые подписи (страны, регионы, города, поселки, деревни, районы, улицы, водоемы, POI) с кириллической локализацией `name:ru` и fallback-вариантом.
  - Настроен источник шрифтов (glyphs) в объекте стиля.
  - В файле `map-view.css` прописаны стили позиционирования контейнера карты.
  - Выделены военные полигоны (military areas): добавлены полупрозрачная красная заливка (`fill-color: #ff4757`), штриховая красная граница (`line-dasharray: [4, 2]`) и отдельный слой текстовых подписей с поддержкой кириллицы для вывода названий полигонов (например, "230 общевойсковой полигон").
  - Оптимизирован масштаб отображения военных подписей: порог отображения (minzoom) снижен до 8 для слоев подписей из полигонов (`landuse`) и точек интереса (`poi`) для гарантированного отображения названий полигонов на обзорных масштабах.
  - Настроен принудительный рендеринг военных подписей: добавлены свойства обхода коллизий (`text-allow-overlap: true`, `text-ignore-placement: true`) для исключения скрытия названий другими элементами. Оформление текста адаптировано под стандарты OSM (курсивный шрифт Noto Sans/Roboto Italic, цвет `#e26d5c` и естественный регистр букв).
  - Исправлена ошибка загрузки глифов (`Error: Unimplemented type: 4`): составные массивы шрифтов в `text-font` заменены на одиночные (`['Noto Sans Regular']` и `['Noto Sans Italic']`), так как статические серверы шрифтов OMT не поддерживают запятые в URL фонтстеков.
  - Настроены подписи дорог: добавлен слой `label_road_ref` для отображения номеров трасс (поле `ref`, например М-1, Р-2) начиная с уровня zoom >= 7, и скорректирован порог видимости названий улиц (`label_road`) с уровня zoom >= 12 для улучшения ориентирования на карте.
  - Карта переведена в светлый режим: изменены все фоновые полигоны (земля `#f4f3f0`, леса `#d0e6d2`, вода `#b0d0f4`), обводки дорог переведены на контрастный белый цвет, а цвета всех текстовых подписей заменены на темные со светлым защитным гало (`#ffffff`, `halo-width: 1.5-2px`) для улучшения читаемости.
- **Интеграция новых слоёв карты и подписей**:
  - Добавлены новые слои: `label_aerodrome` (аэропорты с выводом названия и кода IATA), `label_park` (парки и заповедники), `label_mountain_peak` (горные вершины с отметкой высоты в метрах), `label_housenumber` (номера домов на zoom >= 17), `label_waterway` (подписи рек и ручьёв вдоль линий течения на zoom >= 12).
- **Исправление сборки Tauri в продакшене**:
  - В файле `tauri.conf.json` добавлена директива `"resources"` для автоматической упаковки файла базы данных `assets/belarus.pmtiles` в готовое exe-приложение.
  - В файле `src-tauri/src/lib.rs` обновлен метод `resolve_asset_path()`, реализующий динамическое определение папки ресурсов через API Tauri v2 (`app.path().resource_dir()`) для обеспечения работоспособности упакованного релиза на любых устройствах.
  - Реализован диагностический оверлей на фронтенде для удобного отслеживания логов инициализации карты.
  - Включена хэш-маршрутизация (`HashLocationStrategy`) в Angular для предотвращения сбоев при глубокой адресации путей.
  - Решена проблема `Failed to fetch` на Windows WebView2 путем перевода чтения `.pmtiles` базы данных на IPC-протокол: создана Rust-команда `read_pmtiles_chunk` с поддержкой `seek`/`read` из ресурсов, а во фронтенде написан кастомный класс `TauriPMTilesSource` (наследующий интерфейс `Source` из `pmtiles`) для прямого считывания кусков файла без использования сетевого `fetch`.
  - Установлен пакет `@tauri-apps/api` для осуществления IPC-вызовов (`invoke`).
  - Успешно проведена валидационная сборка приложения (`tauri build`) с формированием MSI и NSIS установщиков.
